19 /** Apply dataflow execution on functions or loops
20 
21  With this mode, Xilinx tools analyze the dataflow dependencies
22  between sequential functions or loops and create channels (based
23  on ping-pong RAMs or FIFOs) that allow consumer functions or loops
24  to start operation before the producer functions or loops have
25  completed.
26 
27  This allows functions or loops to operate in parallel, which
28  decreases latency and improves the throughput.
29 
30  \param[in] f is a function that functions or loops in f will be executed
31  in a dataflow manner.
32 */

34 void dataflow(Functor f) noexcept {

387  /* If there is only 1 shared_ptr user of the buffer, this is the
388  caller of this function, the \c buffer_waiter, so there is no
389  need to get a \ future otherwise there will be a dead-lock if
390  there is only 1 thread waiting for itself.
391

393  for this purpose, it actually increase locally the count by 1,
394  so check for 1 + 1 use count instead...
395  */
396  // If the buffer's destruction triggers a write-back, wait
397  if ((shared_from_this().use_count() > 2) &&
398  modified && (final_write_back || data_host)) {
399  // Create a promise to wait for
400  notify_buffer_destructor = std::promise<void> {};
401  // And return the future to wait for it
402  return notify_buffer_destructor->get_future();
